# 实验动物销售管理系统

本上下文定义实验动物销售管理系统中的核心业务术语，避免客户、订单、区域、票证等概念在需求和架构设计中混用。

## Language

**客户**:
向基地购买实验动物的单位、课题组或联系人组合。客户可以包含多个联系人、多个地址和一个或多个课题组标签。
_Avoid_: 账号、买家

**客户档案**:
销售、后勤和管理者均可查看的客户基础资料，包括单位、课题组标签、联系人、地址、地理区域、结算方式和默认送货设置。客户档案不按销售隔离。
_Avoid_: 私有客户、销售私域客户


**后勤**:
负责配送任务安排、车辆司机信息、出库确认、送达确认和需销售处理标记的系统角色。后勤可读取订单履约所需信息，但不能修改销售域数据。
_Avoid_: 管理员代后勤
**客户归属**:
客户或订单在业务上由某个销售主要跟进的关系。客户档案可共享查看，但订单跟进、价格调整和回款跟进仍应记录具体销售责任人。
_Avoid_: 客户隔离

**配送任务**:
订单确认后从订单派生出来、交给物流后勤安排车辆、司机、配送批次和路线备注的独立业务对象。配送任务承接订单履约信息，但不负责定价、客户档案维护或回款结算；MVP 中默认一个订单生成一个配送任务，并自动进入待安排状态。MVP 中销售和后勤板块完全分开，配送任务通过已出库、已送达等节点同步订单状态。后勤只能维护配送域信息，不能修改客户档案、订单价格、订单明细、结算方式或发票类型。MVP 暂定由后勤在配送任务中确认出库，该动作触发库存扣减和订单出库状态。二期再增加后勤接单和负责人。

**轻审计**:
MVP 只记录不可悄悄发生的高风险业务动作，包括订单改价、订单取消、订单关键状态变化、确认送达、需销售处理标记、库存入库、出库扣减、人工调整批次、票证缺失放行、价格表变更和客户送货地址变更。普通查看、搜索、备注、车辆司机路线备注修改不进入核心审计。
_Avoid_: 全字段审计
**需销售处理**:
后勤在配送任务中发现地址、数量、票证、客户信息等非配送域问题时使用的轻量标记。MVP 不做复杂退回流程；后勤标记问题并填写说明，由销售在订单或客户档案中处理。
_Avoid_: 退回审批、物流驳回
_Avoid_: 物流订单、运单（除非后续接入第三方物流）





**出库扣减**:
配送任务确认出库时对库存批次执行真实扣减的动作。MVP 暂定由系统按优先老化/先进先出推荐扣减批次，后勤确认或调整后生效。
_Avoid_: 自动无确认扣减
**库存预占**:
订单确认后按品系、周龄、性别和数量锁定可售库存的汇总额度。销售订单不绑定具体库存批次；系统内部可以记录预占分配台账，用于取消订单或确认出库后精确释放预占。实际出库批次仍以出库扣减记录为准。
_Avoid_: 批次预占
**改价**:
销售或管理者对未出库、未结算订单的实际成交单价进行人工调整。MVP 中所有销售可改价，但必须填写原因并写入审计日志；已结算订单禁止改价。
_Avoid_: 自动折扣、审批改价
**配送策略提示**:
系统基于订单金额、数量、客户地理区域等条件给销售展示的加量沟通建议。MVP 中配送策略提示不自动改变订单总价，不参与发票和对账金额计算。
_Avoid_: 自动优惠、自动计价
**票证准备**:
订单随货交付所需的合格证和按需发票准备情况。MVP 对票证准备采用弱校验：缺少合格证附件或发票登记时允许出库，但必须提示、填写原因并写入审计日志。
_Avoid_: 强制票证锁单
**地理区域**:
客户在市内配送语境下的区域分类，主要供物流后勤接单后安排车辆和编排路线时参考。地理区域不是自动路线规则，不直接决定固定送货日，也不等同于完整邮政地址。
_Avoid_: 自动路线、销售责任区

## Flagged Ambiguities

**客户隔离**:
早期文档同时出现“销售员互相看不到客户”和“客户列表销售员互相打通”。当前决议：不做客户基础档案隔离，销售与管理者都可查看客户档案；需要记录销售责任人的业务动作另行定义。

## Example Dialogue

销售：订单确认后，后勤从哪里安排车辆？

管理者：订单确认后生成配送任务。配送任务带出客户地理区域和详细送货地址，后勤在配送任务里安排车辆、司机、配送批次和路线备注。

销售：客户不是我建的，我能看吗？

管理者：能看客户档案。谁负责这笔订单、谁改过价格、谁跟进回款，要在订单和审计记录里体现。
