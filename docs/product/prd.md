## 1. 产品概述

### 1.1 背景

实验动物养殖基地目前销售管理依赖人工+微信沟通，存在以下问题：

- 库存周龄靠人工记忆，超龄未售无法自动提醒
- 客户信息散落在销售员个人微信中，无法共享查询
- 订单状态无法追踪（哪些待确认/已出库/已开票/已结算？）
- 月结应收账款缺少系统化管理

### 1.2 产品定位

**内部Web管理工具**，供基地销售人员和管理者使用。不直接面向客户（客户继续通过微信与销售沟通，销售将信息录入系统）。

### 1.3 经营品类

| 品类  | 等级   |
| --- | ---- |
| 小鼠  | SPF级 |
| 大鼠  | SPF级 |
| 豚鼠  | 普通级  |
| 兔子  | 普通级  |

- 周龄范围：0周起售，超过8周需老化提醒
- 周龄更新：每周日自动+1
- 品系数量：待朋友提供品系价目表后确定

## 2. 用户角色

> 本节已按 [framework.md](../architecture/framework.md) 架构冻结决策回写，角色范围以 [framework.md §8 权限矩阵](../architecture/framework.md#8-权限矩阵) 为准。

| 角色   | 人数   | 权限范围                                  |
| ---- | ---- | ------------------------------------- |
| 销售员  | ≤8人  | 客户档案（共享，可编辑）、创建/确认订单、改价、票证归档、结算标记     |
| 后勤   | 视实际配置 | 配送任务（车辆/司机/批次）、确认出库、确认送达、标记需销售处理、票证缺失放行 |
| 管理者  | ≤2人  | 全量数据查看、统计看板、价格管理，销售与后勤权限均具备           |
| 最多并发 | ≤10人 | 互联网访问（支持居家办公）                          |

### 2.1 客户档案访问规则

客户基础档案**不做销售隔离**：销售、后勤、管理者均可查看；销售和管理者可编辑，后勤只读（见 [ADR-0003](../adr/0003-use-shared-customer-records-without-sales-isolation.md)）。客户送货地址变更需轻审计。订单仍归属创建它的销售员，但客户可见性不受此限制。

## 3. 核心业务流程

### 3.1 主流程

> MVP 中销售与后勤分属两个独立工作域，订单确认后自动派生配送任务；出库/送达由后勤在配送任务中确认，而非销售直接操作（见 [ADR-0002](../adr/0002-separate-sales-orders-from-delivery-tasks.md)）。

```
客户微信询价
    │
    ▼
销售查库存 + 报价（查系统价格表/合同价）
    │
    ▼
客户确认采购
    │
    ▼
销售创建订单（选品系+周龄+性别+数量+送货方式+开票类型）
    │  订单状态: pending（待确认）
    │  订单编号: XS+年月日+8位随机hex
    ▼
销售确认订单 → 状态: confirmed（已确认）→ 库存汇总预占
    │  系统自动创建配送任务（pending_schedule）
    ▼
按客户需要提前准备发票；每批次提前准备合格证（随货交付）
    │
    ▼
后勤安排车辆/司机/批次 → 配送任务: scheduled
    │
    ▼
送货日，后勤确认出库（系统推荐批次，后勤确认扣减）
    → 配送任务: shipped → 订单状态同步: shipped（已出库）→ 库存真正扣减
    │
    ▼
后勤确认送达（动物+合格证+按需发票随货到达）
    → 配送任务: delivered → 订单状态同步: delivered（已送达）
    │
    ▼
销售上传合格证附件；登记发票信息（按客户需要）→ 状态: invoiced（已开票/票证已归档）
    │
    ▼
销售登记回款 → 状态: settled（已结算）
```

### 3.2 取消流程

任意非终态节点均可取消 → 状态: cancelled → 释放预占库存

### 3.3 送货节奏

| 送货方式 | 说明            |
| ---- | ------------- |
| 一三五  | 固定路线，周一/三/五配送 |
| 二四   | 固定路线，周二/四配送   |
| 闪送   | 基地叫闪送，费用客户承担  |
| 自提   | 客户自行取货        |

- 每位客户可设默认送货方式，单笔订单可覆盖
- 客户信息需划分地理区域（如城市/区县/园区/校区），用于默认匹配送货路线
- 送货区域可配置默认节奏（如某区域走一三五，某区域走二四），单笔订单可覆盖
- 系统可配置送货策略提示，例如订单金额或数量达到阈值后免纸箱运费，便于销售与客户进行加量沟通
- 送货区域/路线及策略阈值待朋友提供

## 5. 定价模型

### 5.1 定价维度

- 一级维度：品系 + 周龄
- 雌雄同价
- 体重：一期不做（偶发需求，转人工处理）

### 5.2 定价规则

| 场景      | 规则                  |
| ------- | ------------------- |
| 标准定价    | 品系+周龄 → 查当前有效价格表    |
| 合同价/协议价 | 有大客户协议的，销售员手动录入实际单价 |
| 特批改价    | 销售员可直接修改单价          |
| 阶梯价/折扣  | 不自动处理，转人工           |
| 预存款客户   | 转人工处理               |

### 5.3 价格变更

- 价格支持调价（价格变化不大但需有此功能）
- 价格表带生效日期，调价不影响历史订单

### 5.4 配送费用与加量策略

- 系统支持配置配送费用策略，用于辅助销售报价和客户加量沟通
- 策略示例：订单金额满 X 元免纸箱运费；订单数量满 X 只免纸箱运费；指定区域/路线适用不同阈值
- 策略命中时，订单页提示“已满足免纸箱运费条件”
- 策略未命中但接近阈值时，订单页提示“再增加 X 元/数量可免纸箱运费”
- 一期先做策略提示和人工确认，不自动强制改价或自动生成优惠

## 6. 开票与合格证

| 事项   | 处理方式                       |
| ---- | -------------------------- |
| 合格证  | 每批次必开，由官方系统提前开具，随货交付；本系统上传扫描件作为附件 |
| 发票   | 按客户需要（部分要部分不要），提前设置客户标签；需要开票的订单发票随货交付 |
| 开票类型 | 默认技术服务费/耗材，客户级别设定，单笔订单可覆盖  |

### 12.3 B1-B3 解阻塞方案

这三个是 **🔴 阻塞项**，没有它们无法进行任何有意义的数据初始化：

**B1 品系价目表 — 最小推进方案：**

请朋友至少提供 3-5 个最常用品系的价格样例，格式如下：

```
品系名, 品类, 周龄, 单价
C57BL/6, 小鼠, 3周, XX元
C57BL/6, 小鼠, 4周, XX元
...
```

有了样例就能建好 price_list 表结构和逻辑，剩余品系后续批量导入即可。

**B2 送货区域 — 最小推进方案：**

请朋友口述或微信发一段话即可，不需要正式文档。例如："北区是海淀+朝阳+昌平，走一三五；南区是丰台+大兴，走二四"。同时确认客户地理区域的划分口径（城市/区县/园区/校区）和默认路线。

**B3 品系清单 — 可推导：**

如果朋友提供价目表（B1），品系清单即可自动提取。因此 **B1 解决后 B3 自动消除**。

## 13. 术语表

| 术语     | 说明                          |
| ------ | --------------------------- |
| 品系     | 实验动物的遗传学分类（如C57BL/6、BALB/c） |

| 月结     | 按账期定期结算（非当场付清）              |
| 合格证    | 《实验动物质量合格证》，官方系统开具          |
